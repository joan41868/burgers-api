LimeBurger API

Common info

Rate Limiting - IP based, 15 requests/min

CORS compliant

Root endpoint: https://limeburger-api.herokuapp.com/ (here)

GET endpoints:

  • Get by ID

    /burgers/{id} Where {id} is an integer, representing the ID of the burger.

    Returns a single burger, like:

    {"name": "LimeBurger1", "image_url":"https://limeburger-api.herokuapp.com/images/limeburger.jpg", "id":5}

  • Get with queries

    /burgers?burger_name={name}&pageNum={pageN}&perPage={perPageN} Where burger_name is the burger name(or like it), pageNum and perPage are integers

POST endpoints:

  • Create burger

    /burger

    Accepts burgers, like:

    {"name": "LimeBurger1", "image_url":"https://limeburger-api.herokuapp.com/images/limeburger.jpg"}

    Returns the created burger, like:

    {"name": "LimeBurger1", "image_url":"https://limeburger-api.herokuapp.com/images/limeburger.jpg", "id":5}

Pagination

Pagination is implemented via offset/limit SQL queries.

The default pageNumber is 1, and the items on the page (perPage param) are 15.

Request query parameters:

burger_name - a string, which is contained in the desired burger's name.

perPage - an integer, denoting how many items you want to see per page.

pageNum- an integer, representing the page you want to see.
